Plan Re-evaluation

Assessment of Your Plan

Your architecture is solid — the loop (observe → understand → plan → execute → verify → memory) is exactly how production computer-use agents work. The layer separation is clean. A few refinements will save you significant pain early on.

Keep

  • Core runtime loop design
  • 5-layer separation
  • Claude as the vision+planning LLM
  • PyAutoGUI + Playwright combo
  • Redis for session memory
  • FastAPI for the control API

Revise / Simplify

  • Drop OmniParser initially (heavy setup)
  • Skip Qdrant in Phase 1 (overkill)
  • Drop LangGraph for now (use raw loop)
  • Supermemory → Redis only first
  • PaddleOCR → only if Claude vision fails
  • UI-TARS → Phase 3+ addition

# main.py — the runtime loop async def run_agent(goal: str, max_steps: int = 20): history = [] while len(history) < max_steps: screenshot = await capture_screen() ui_state = await parse_ui(screenshot) action = await plan(goal, ui_state, history) if action.type == "done": return {"status": "success", "steps": len(history)} before_state = ui_state await execute(action) await asyncio.sleep(0.8) # wait for UI to settle new_screenshot = await capture_screen() new_state = await parse_ui(new_screenshot) verified = await verify(action, before_state, new_state) history.append({"action": action, "verified": verified})
